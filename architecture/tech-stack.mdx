## Overview

The DIDG System is built with modern web technologies, prioritizing developer experience, type safety, and performance. The stack is carefully curated to provide a robust foundation for an academic management platform.

## Core Framework

### Next.js 14.1.0

The application is built on **Next.js 14**, utilizing the App Router architecture.

**Key Features Used:**
- **App Router**: File-system based routing with layouts
- **React Server Components**: Server-side rendering by default
- **Server Actions**: Type-safe mutations without API routes
- **Streaming SSR**: Progressive page rendering
- **Image Optimization**: Automatic image optimization
- **Font Optimization**: Automatic font optimization with Google Fonts

**Configuration** (`package.json:26`):
```json
"next": "14.1.0"
```

<Note>
Next.js 14 introduced stable Server Actions, which are heavily utilized throughout the DIDG System for all data mutations.
</Note>

### React 18

**React 18** provides the foundation for the UI layer with concurrent rendering features.

**Dependencies** (`package.json:28-29`):

## Backend & Database

### Supabase

Supabase provides the entire backend infrastructure.

**Dependencies** (`package.json:16-17`):
```json
"@supabase/ssr": "^0.1.0",
"@supabase/supabase-js": "^2.39.3"
```

**Supabase CLI** (dev dependency) (`package.json:49`):
```json
"supabase": "^2.71.0"
```

**Services Used:**

1. **PostgreSQL Database**: Primary data storage
2. **Authentication**: User management with RLS
3. **Storage**: File storage for course materials (`src/core/actions/ayudantias.ts:34`)
4. **Row Level Security**: Fine-grained access control

**Client Architecture:**

- **Browser Client** (`src/infrastructure/supabase/client.ts:4`): For client-side operations
- **Server Client** (`src/infrastructure/supabase/server.ts:6`): For server-side operations with cookies
- **Admin Client** (`src/infrastructure/supabase/admin.ts:5`): For operations requiring elevated privileges

### Database

**PostgreSQL** (via Supabase) with the following features:

- Auto-generated TypeScript types
- Foreign key constraints with CASCADE deletes
- Row Level Security policies
- Triggers and functions

See [Database Schema](/architecture/database-schema) for detailed schema documentation.

## Environment Variables

The application requires the following environment variables:

```bash
# Supabase
NEXT_PUBLIC_SUPABASE_URL=
NEXT_PUBLIC_SUPABASE_ANON_KEY=
SUPABASE_SERVICE_ROLE_KEY=

# Telegram
TELEGRAM_BOT_TOKEN=
TELEGRAM_CHAT_ID=

# Application
NEXT_PUBLIC_SITE_URL=
```

<Note type="warning">
**Security**: The `SUPABASE_SERVICE_ROLE_KEY` should never be exposed to the client. It is only used in server-side code (Server Actions and Server Components).
</Note>
