# Configuration Guide

This guide covers all configuration required to run DIDG System locally, including environment variables, Supabase setup, and framework configurations.

## Environment Variables

The DIDG System uses environment variables to manage sensitive credentials and configuration settings.

### Required Environment Variables

<Steps>
  <Step title="Create .env.local file">
    Create a `.env.local` file in the root of your project:

    ```bash
    cd didg-website
    touch .env.local
    ```

    <Warning>
      Never commit `.env.local` to version control. It should be listed in `.gitignore`.
    </Warning>
  </Step>

  <Step title="Configure Supabase Variables">
    Add your Supabase project credentials:

    ```bash .env.local
    # Supabase Configuration
    NEXT_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
    NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-key-here
    SUPABASE_SERVICE_ROLE_KEY=your-service-role-key-here
    ```

    **Where to find these values:**

    1. Go to [Supabase Dashboard](https://supabase.com/dashboard)
    2. Select your project
    3. Navigate to **Settings** → **API**
    4. Copy the following:
       - **URL** → `NEXT_PUBLIC_SUPABASE_URL`
       - **anon/public** key → `NEXT_PUBLIC_SUPABASE_ANON_KEY`
       - **service_role** key → `SUPABASE_SERVICE_ROLE_KEY`

    <Info>
      Variables prefixed with `NEXT_PUBLIC_` are exposed to the browser. The service role key is server-side only and has elevated privileges.
    </Info>
  </Step>

  <Step title="Configure Telegram Bot (Optional)">
    Add Telegram notification credentials:

    ```bash .env.local
    # Telegram Bot Configuration (Optional)
    TELEGRAM_BOT_TOKEN=1234567890:ABCdefGHIjklMNOpqrsTUVwxyz
    TELEGRAM_CHAT_ID=987654321
    ```

    **Get these values:**
    - `TELEGRAM_BOT_TOKEN`: From [@BotFather](https://t.me/botfather) when creating your bot
    - `TELEGRAM_CHAT_ID`: Your personal Telegram chat ID (see [Environment Setup](/development/environment-setup#set-up-telegram-bot-optional))

    **Used for:**
    - Student registration notifications
    - Grade upload alerts
    - System event notifications

    <Tip>
      If these variables are not set, the system will log a warning but continue to function without Telegram notifications.
    </Tip>
  </Step>

  <Step title="Configure Site URL (Optional)">
    Set your site URL for metadata and SEO:

    ```bash .env.local
    # Site Configuration
    NEXT_PUBLIC_SITE_URL=http://localhost:3000
    ```

    **Defaults to:** `https://www.danielduran.engineer` if not set

    Change this when deploying to production.
  </Step>
</Steps>

## Supabase Project Configuration

### Setting Up Your Supabase Project

<Steps>
  <Step title="Create Supabase Project">
    If you haven't already:

    1. Go to [Supabase Dashboard](https://supabase.com/dashboard)
    2. Click **New Project**
    3. Fill in:
       - **Name**: DIDG Development (or your preferred name)
       - **Database Password**: Strong password (save this!)
       - **Region**: Choose closest to your location
    4. Click **Create new project**

    Wait 2-3 minutes for provisioning to complete.
  </Step>

  <Step title="Configure Storage Buckets">
    DIDG System uses Supabase Storage for file uploads.

    **Create the materials bucket:**

    1. Go to **Storage** in your Supabase dashboard
    2. Click **Create a new bucket**
    3. Name: `materials`
    4. Make it **public** (for student access to course materials)
    5. Click **Create bucket**

    **Bucket policies are managed in source:** `src/infrastructure/supabase/`
  </Step>

  <Step title="Set Up Database Schema">
    Database migrations will be handled when you run the development server.

    See [Running Locally](/development/running-locally#database-setup) for migration instructions.
  </Step>
</Steps>

## Next.js Configuration

The project's Next.js configuration is defined in `next.config.js`:

```javascript next.config.js
/** @type {import('next').NextConfig} */
const nextConfig = {
  images: {
    remotePatterns: [
      {
        protocol: "https",
        hostname: "**.supabase.co", // Supabase Storage files
      },
      {
        protocol: "https",
        hostname: "github.com", // GitHub profile images
      },
      {
        protocol: "https",
        hostname: "avatars.githubusercontent.com", // GitHub avatars
      },
      {
        protocol: "https",
        hostname: "**.imgur.com", // Imgur images
      },
    ],
  },
};

module.exports = nextConfig;
```

### Image Optimization

The `remotePatterns` configuration allows Next.js Image Optimization for:
- **Supabase Storage** (`**.supabase.co`) - Course materials and uploaded files
- **GitHub** - Profile pictures and avatars
- **Imgur** - External images

<Warning>
  If you need to allow images from additional domains, add them to the `remotePatterns` array in `next.config.js`.
</Warning>

## TypeScript Configuration

The project's TypeScript settings are in `tsconfig.json`:

```json tsconfig.json
{
  "compilerOptions": {
    "target": "ES2017",
    "lib": ["dom", "dom.iterable", "esnext"],
    "allowJs": true,
    "skipLibCheck": true,
    "strict": true,
    "noEmit": true,
    "esModuleInterop": true,
    "module": "esnext",
    "moduleResolution": "bundler",
    "resolveJsonModule": true,
    "isolatedModules": true,
    "jsx": "preserve",
    "incremental": true,
    "plugins": [
      {
        "name": "next"
      }
    ],
    "paths": {
      "@/*": ["./src/*"]
    }
  },
  "include": [
    "next-env.d.ts",
    "**/*.ts",
    "**/*.tsx",
    ".next/types/**/*.ts",
    ".next/dev/types/**/*.ts",
    "**/*.mts"
  ],
  "exclude": ["node_modules"]

```

### Path Aliases

The `@/*` path alias maps to `./src/*`, allowing clean imports:

```typescript
// Instead of:
import { Button } from '../../../components/ui/button'

// Use:
import { Button } from '@/components/ui/button'
```

<Tip>
  All imports from the `src` directory should use the `@/` alias for consistency.
</Tip>

## Supabase Client Configuration

The project uses different Supabase clients for different contexts:

### Server-Side Client

**File:** `src/infrastructure/supabase/server.ts`

Used in Server Components and Server Actions:

```typescript
import { createClient } from '@/infrastructure/supabase/server'

export async function ServerComponent() {
  const supabase = await createClient()
  const { data } = await supabase.from('students').select('*')
  // ...
}
```

### Client-Side Client

**File:** `src/infrastructure/supabase/client.ts`

Used in Client Components:

```typescript
import { createClient } from '@/infrastructure/supabase/client'

export function ClientComponent() {
  const supabase = createClient()
  // ...
}
```

### Admin Client

**File:** `src/infrastructure/supabase/admin.ts`

Used for privileged operations (uses `SUPABASE_SERVICE_ROLE_KEY`):

```typescript
import { createAdminClient } from '@/infrastructure/supabase/admin'

export async function uploadGrades() {
  const supabase = createAdminClient()
  // Bypass RLS policies
}
```

<Warning>
  The admin client bypasses Row Level Security (RLS). Only use it in server-side code for trusted operations.
</Warning>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Supabase connection errors">
    If you see `Invalid API key` or connection errors:

    1. Verify your `.env.local` has correct values
    2. Check that variable names match exactly (case-sensitive)
    3. Restart your dev server after changing `.env.local`
    4. Ensure Supabase project is not paused (check dashboard)
  </Accordion>

  <Accordion title="Image optimization errors">
    If images fail to load:

    1. Check the image hostname is in `next.config.js` `remotePatterns`
    2. For Supabase Storage, ensure the bucket is public
    3. Verify the image URL is correctly formatted
  </Accordion>

  <Accordion title="Telegram notifications not working">
    If Telegram messages aren't sending:

    1. Verify `TELEGRAM_BOT_TOKEN` and `TELEGRAM_CHAT_ID` in `.env.local`
    2. Start a conversation with your bot first
    3. Check bot token is valid (no spaces or line breaks)
    4. Check server logs for error messages
  </Accordion>

  <Accordion title="TypeScript path alias not working">
    If `@/` imports show errors:

    1. Restart your TypeScript server in VS Code: `Cmd+Shift+P` → "TypeScript: Restart TS Server"
    2. Verify `tsconfig.json` includes the `paths` configuration
    3. Ensure the file you're importing from is in the `src` directory
  </Accordion>
</AccordionGroup>
