# Dashboard Components

The DIDG System dashboard provides a comprehensive suite of components for academic management. All components follow a theme-aware design pattern using CSS variables for light/dark mode support.

## Component Hierarchy

```
Dashboard
├── Sidebar (Navigation)
├── StudentsTable (Student Management)
├── StudentRegisterForm (Student Registration)
├── UploadGradesForm (Grade Import)
├── AdminResourceList (Resource Management)
│   └── ResourceFormModal
├── AdminAyudantiaList (Class Session Management)
│   └── AyudantiaCard
│       └── AyudantiaForm
├── MessageActions (Message Interactions)
├── RecentDownloads (Analytics Widget)
└── AnalyticsChart (Data Visualization)
```

## StudentsTable

**Location:** `~/workspace/source/didg-website/src/components/dashboard/StudentsTable.tsx`

### Props

```typescript
interface Props {

}
```

### Features

- Inline edit modal with validation
- Email/password change confirmation flow
- Optimistic UI updates
- Security warning for credential changes
- Delete functionality with confirmation

### Usage Example

```tsx
import { StudentsTable } from "@/components/dashboard/StudentsTable";
import { createClient } from "@/infrastructure/supabase/server";

export default async function StudentsPage() {
  const supabase = await createClient();
  const { data: students } = await supabase
    .from("profiles")
    .select("*")
    .eq("role", "student")
    .order("created_at", { ascending: false });

  return <StudentsTable students={students || []} />;
}
```

### Usage Example

```tsx

export default async function DashboardPage() {

    </div>

}
```

### Time Formatting Logic

```typescript
const formatTimeAgo = (dateString: string) => {
  const diffInSeconds = Math.floor((now.getTime() - date.getTime()) / 1000);
  if (diffInSeconds < 60) return "Hace un momento";
  if (diffInSeconds < 3600) return `Hace ${Math.floor(diffInSeconds / 60)} min`;
  if (diffInSeconds < 86400) return `Hace ${Math.floor(diffInSeconds / 3600)} h`;
  return date.toLocaleDateString();
};
```

## AnalyticsChart

Bar chart visualization for download analytics using Recharts.

**Location:** `~/workspace/source/didg-website/src/components/dashboard/AnalyticsChart.tsx`

### Props

```typescript
interface Props {
  data: { name: string; value: number }[];
}
```

### Features

- Displays top 6 items + "Otros" category
- Auto-sorts by value
- Theme-aware colors using CSS variables
- Responsive container
- Tooltip with custom styling

### Usage Example

```tsx
import { AnalyticsChart } from "@/components/dashboard/AnalyticsChart";

export default async function DashboardPage() {
  const { data: stats } = await supabase.rpc("get_download_stats");

  return (
    <div className="bg-surface border border-text-main/10 rounded-xl p-6">
      <h3 className="text-lg font-bold mb-4">Descargas por Recurso</h3>
      <AnalyticsChart data={stats || []} />
    </div>
  );
}
```

### Data Processing

```typescript
const processData = () => {
  const sortedData = [...data].sort((a, b) => b.value - a.value);
  const topItems = sortedData.slice(0, MAX_ITEMS);
  const otherItems = sortedData.slice(MAX_ITEMS);
  const otherValue = otherItems.reduce((sum, item) => sum + item.value, 0);

  if (otherValue > 0) {
    topItems.push({ name: "Otros", value: otherValue });
  }

  return topItems;
};
```

## Dark Mode Support

All components automatically adapt to theme changes via:

1. **CSS Variables** - Dynamic color tokens
2. **Tailwind Classes** - `bg-surface`, `text-text-main`, etc.
3. **ThemeProvider** - From `next-themes`

No manual dark mode classes needed - the design system handles it automatically.
